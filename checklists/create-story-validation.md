# Create Story — Quality Validation Checklist

## Source Analysis
- [ ] Epics file loaded and target story requirements extracted
- [ ] Architecture file loaded — tech stack, patterns, structure identified
- [ ] Previous story loaded (if story_num > 1) — learnings extracted
- [ ] PRD consulted for acceptance criteria detail

## Story Quality
- [ ] User story follows "As a / I want / So that" format
- [ ] All acceptance criteria are testable (Given/When/Then)
- [ ] Tasks/subtasks map to acceptance criteria
- [ ] Tasks are ordered logically for implementation

## Dev Notes Quality
- [ ] Architecture patterns and constraints documented
- [ ] Specific files/directories to modify identified
- [ ] Testing strategy clear (frameworks, patterns)
- [ ] Dependencies and libraries with versions specified
- [ ] Previous story learnings incorporated
- [ ] No vague or ambiguous instructions

## Disaster Prevention
- [ ] No duplicate functionality — checked existing code/patterns
- [ ] Correct libraries and framework versions specified
- [ ] File locations match project structure
- [ ] No breaking changes to existing functionality
- [ ] UX requirements from design spec included
